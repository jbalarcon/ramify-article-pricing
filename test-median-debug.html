Debug Median Cost Calculation

Problem Identified

⚠️ Default Configuration Issue

Both baseline and simulation start with identical default configurations:

baseline: { model: 'PW', params: { rate: 0.13 }, bonusPercent: 0 } simulation: { model: 'PW', params: { rate: 0.13 }, bonusPercent: 0 }

This means if you don't change the configuration, all costs will be identical!

Testing Steps

  1. Load the application and upload sample-data.csv
  2. Check initial values: Median costs should be identical if configuration hasn't been changed
  3. Open configuration modal
  4. Change the simulation model to something different:
    • Try Fixed Price (FP) with amount 150€
    • Or change PW rate to 0.15€ instead of 0.13€
    • Or add a bonus percentage
  5. Apply configuration
  6. Check that median values are now different

Expected Behavior

  • ✓ When configurations are identical: Median baseline = Median simulation
  • ✓ When configurations differ: Median baseline ≠ Median simulation
  • ✓ Average costs should follow the same pattern

How to Verify in Console

Open browser console (F12) and after loading data, run:

// Get the simulator instance const sim = window.pricingSimulator; // Check current configuration console.log('Baseline config:', sim.configuration.globalDefaults.baseline); console.log('Simulation config:', sim.configuration.globalDefaults.simulation); // Check if costs are different const baselineCosts = sim.articles.map(a => a.baselineCost); const simulationCosts = sim.articles.map(a => a.simulationCost); console.log('First 5 baseline costs:', baselineCosts.slice(0, 5)); console.log('First 5 simulation costs:', simulationCosts.slice(0, 5)); // Check if they're identical const areIdentical = baselineCosts.every((cost, i) => cost === simulationCosts[i]); console.log('Costs are identical?', areIdentical);

Solution

The median calculation is working correctly. The issue is that both configurations start with the same values.

To see different median values:

  1. Open the configuration modal
  2. Change the "Nouvelle Tarification" (simulation) to use different parameters
  3. Apply the configuration
  4. The median values will then be different
